AI Semantic Cache

What is Semantic Caching?

Semantic caching is a technique for caching LLM (Large Language Model) responses based on the meaning of a prompt rather than its exact text. Traditional caches use exact string matching — "What is Python?" and "Explain Python" would be treated as completely different keys. A semantic cache recognizes these prompts are asking roughly the same thing and returns the cached response.

In production, this typically uses embedding vectors and cosine similarity to measure how close two prompts are. This demo simulates that behavior using token normalization and fingerprint matching — no ML dependencies required.

How It Works

The normalization pipeline transforms raw prompts into canonical fingerprints:

{{ flow_diagram(["Raw Prompt", "Lowercase", "Remove Punctuation", "Strip Stopwords", "Sort Tokens", "SHA-256 Fingerprint"], "SHA-256 Fingerprint") }}
  1. Normalize: The prompt is lowercased, punctuation is removed, common stopwords are stripped, and remaining tokens are sorted alphabetically.
  2. Fingerprint: The normalized string is hashed with SHA-256, creating a canonical identifier.
  3. Exact match: If the fingerprint matches a stored entry, return it immediately.
  4. Semantic match: Generate n-1 gram variants (dropping each token once) and check if any variant's fingerprint matches the index.
  5. Miss: If no match, call the LLM, store the response, and index its variants for future matching.
