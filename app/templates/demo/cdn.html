CDN Cache

What is CDN Caching?

A Content Delivery Network (CDN) is a globally distributed network of edge servers that cache content close to end users. Instead of every request traveling to your origin server (which might be in one data center), the CDN serves cached copies from the nearest Point of Presence (PoP) — often just a few milliseconds away.

CDNs sit between the browser and your origin server. On the first request, the CDN fetches from origin (a cache fill), stores the response, and serves subsequent requests from its edge cache. This dramatically reduces latency for global users and offloads traffic from your origin infrastructure.

How It Works

CDN caching adds an edge layer between the client and your origin server:

{{ flow_diagram(["User (Tokyo)", "CDN Edge (Tokyo)", "CDN Edge (US)", "Origin Server (US)"], "CDN Edge (Tokyo)") }}
  1. DNS resolution: The user's request is routed to the nearest CDN edge node via Anycast or DNS-based routing.
  2. Edge cache check: The edge server checks if it has a cached copy that hasn't expired.
  3. Cache HIT: If found, the edge responds directly — the request never reaches your origin.
  4. Cache MISS (fill): The edge fetches from origin (or a mid-tier shield), caches the response, and serves it.
  5. Cache purge: When content changes, you can invalidate the CDN cache via API, forcing a fresh fill on the next request.
