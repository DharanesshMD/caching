What is Caching?

Caching stores copies of data in a faster-access location so future requests can be served more quickly. Instead of computing results or fetching from a slow data source every time, a cache returns previously stored answers in milliseconds. Every layer of a modern web stack uses caching — from the user's browser all the way down to the database engine.

The Caching Stack

Requests flow through multiple caching layers. Each layer intercepts the request and can return a cached response, avoiding the slower layers below.

Explore the Layers

🌐

Browser Cache

Client-side caching using HTTP headers like Cache-Control, ETag, and 304 Not Modified responses.

 🌍

CDN Cache

Edge nodes distributed globally that cache content close to users, reducing latency and origin load.

 🖥️

Server / Redis Cache

Centralized server-side caching with Redis — TTL, serialization, and shared state across instances.

 ⚡

In-Memory Cache

Process-local caching for ultra-fast access — LRU eviction, lazy expiration, and zero network overhead.

 🗄️

Database Cache

Query-level caching that avoids repeated expensive database queries using write-through and cache-aside patterns.

 🤖

AI Semantic Cache

Intelligent caching for LLM responses — matches semantically similar prompts to save cost and latency.
